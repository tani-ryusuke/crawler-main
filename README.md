# クローラーロボット開発プロジェクト（ファームウェア側）

本リポジトリは、ESP32-S3マイコンおよびESP-IDF（v5.3.5）開発環境を用いた、3層構造クローラーロボットのメイン制御ファームウェアです。FreeRTOSによる定周期タスク管理のもと、ESP-NOWを用いた低遅延ワイヤレス制御と、実機ハードウェアの破壊を防ぐための多重のソフトウェア安全機構を統合しています。

## 主な実装機能と技術的特徴

### 1. FreeRTOSによる20ms定周期のソフトスタート制御
メインループを20ms周期で厳密に管理し、モーター出力を目標値へ滑らかに近づける傾き制御を実行。DCモーター起動時の過大な突入電流を抑制し、バッテリーの電圧降下やロジック系のリセットを防ぎます。

### 2. 通信品質のリアルタイムLED可視化
20msのタイマーコールバックを用いて直近のデータ受信成功率を算出。スマートLED（WS2812）のRGB値を滑らかに合成し、通信品質を緑から赤へとリアルタイムにグラデーション変化させることで、実機評価時のデバッグ性を向上させています。

### 3. 確定的なチャタリング対策
システム時間に基づく300msの不感時間管理を実装。物理ボタンのチャタリングによる誤動作を排除し、意図したタイミングでのみ緊急停止モードが反転する立ち上がりエッジ検出ロジックを構築しています。

### 4. ハードウェア保護を最優先したモーター制御
Hブリッジ回路における上下アームの同時ON（貫通電流）や急反転時の逆起電力によるFET破壊を防ぐため、対向アームの出力が0になるまで逆方向への出力を待機させるソフトウェア・インターロックを実装。さらに、超信地旋回入力を自動検知して最高出力を一律50%に自動制限するギヤ保護機能や、緊急停止時に約0.04秒で電力をカットオフする動的減速ステップ切り替えを搭載しています。

### 5. 信頼度メーター方式によるPIRセンサー制御とモード管理
PIRセンサーの入力値に対して独自の「信頼度メーター（0〜10）」によるヒステリシス処理（5以上で検知ON、2以下で検知OFF）を導入し、チャタリングやチラつきを対策。通常走行モード（`MODE_DRIVE`）とアタッチメント操作モード（`MODE_ATTACHMENT`）の厳格な状態管理を行い、アタッチメントモード中は走行出力を強制的にゼロに固定する安全ガードを搭載しています。

### 6. 格闘ゲームコマンド判定と非同期PWMブザー演奏
コントローラーから送出されるテンキー方向（1〜9）の入力履歴をリングバッファで常時解析し、波動拳（テトリス）、昇竜拳（ダース・ベイダーのテーマ）、竜巻旋風脚（パックマン）といった格闘ゲームのコマンド成立を判定。FreeRTOSのバックグラウンドタスクによる非同期PWMブザー演奏や、有線UART通信（115200 bps）を通じたカメラマイコン（XIAO）へのデータ中継・警報通知を行います。
